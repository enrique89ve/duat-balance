# DUAT Snapshot

Website for viewing and querying **DUAT** token balances from the Ragnarok game on the Hive blockchain.

## Requirements

- [Bun](https://bun.sh/) v1.0+

## Installation

```bash
bun install
```

## Commands

```bash
bun run collect      # Collect snapshot data (balances + claims)
bun run dev          # Development server with hot reload
bun run start        # Production server
bun run type-check   # Type checking (tsc --noEmit)
```

## How data is extracted

The collection process (`bun run collect`) fetches data from two external sources and consolidates it into a single JSON file:

### 1. DUAT node state

Connects to the active Duat node (`http://121.99.241.161:5306`) and queries:

- **`GET /state`** — Full protocol state. Fields extracted:
  - `state.balances` — Account -> liquid balance map (raw integers)
  - `state.pow` — Account -> staked tokens (Powered Up)
  - `state.gov` — Account -> governance-locked tokens
  - `state.snap` — Account -> airdrop snapshot balance
  - `state.stats.tokenSupply` — Total token supply

- **`GET /api/coin_detail`** — Token metadata:
  - `symbol` — Token symbol (DUAT)
  - `precision` — Decimal places (3 = divide raw by 1000)

The **total balance** for each holder is calculated as:

```
total = balance (liquid) + poweredUp (staked) + gov (governance)
```

All values are stored as **raw integers**. To get the real value, divide by `10^precision`. Example: a raw balance of `9896316` with precision 3 equals `9,896.316 DUAT`.

### 2. Claim operations (HafSQL)

Queries the public HafSQL API (`https://api.syncad.com`) to fetch all `custom_json` transactions with ID `duat_drop_claim`:

```
GET /hafsql/operations/custom_json/duat_drop_claim?start={block}&limit=-1000
```

Collection is paginated (batches of 1000 operations) starting from block `63,640,800` (first known claim). The claiming account is extracted from each operation.

### 3. Consolidation

Both data sources are merged into the snapshot file (`data/duat-snapshot.json`):

- **Holder entries**: All accounts found across balances, gov, pow, snap, and claims are combined. System-reserved accounts (`ra`, `rc`, `rd`, `ri`, `rn`, `rm`) are excluded. Each entry includes:
  - Broken-down balances (liquid, staked, governance, total)
  - Whether the airdrop was claimed (`claimed`)
  - Original airdrop snapshot balance (`snapBalance`)
  - Whether tokens were acquired via DEX without a claim (`acquiredViaDex`)

- **Claim analysis**: Aggregated airdrop statistics (retention rate, claimers, unclaimed tokens, etc.)

- **Token stats**: Global token metrics (supply, distribution, node, protocol version)

## Architecture

```
index.ts                   -> Entry point (imports the server)
src/server.ts              -> HTTP server (Bun.serve) + static file serving
src/collector.ts           -> Data collection script
src/lib/duatNode.ts        -> node client (state, balances)
src/lib/hafsql.ts          -> HafSQL client (claim operations)
src/lib/types.ts           -> TypeScript domain types
src/lib/utils.ts           -> Shared backend utilities
data/duat-snapshot.json    -> Collected snapshot (generated by collect)
public/index.html          -> Main page (balance tracker)
public/info.html           -> Analytics page (claims, top holders)
public/styles.css          -> Styles
public/app.js              -> Balance tracker logic
public/info.js             -> Analytics logic
public/utils.js            -> Shared frontend utilities
public/cache.js            -> ETag-based localStorage cache
```

## Stack

- **Runtime**: Bun
- **Backend**: Strict TypeScript, `Bun.serve()`
- **Frontend**: Vanilla HTML/CSS/JS (no frameworks)
- **Zero external dependencies** (only `@types/bun` and `typescript` as dev)

## Server API

| Endpoint                  | Description                       |
| ------------------------- | --------------------------------- |
| `GET /api/snapshot-data`  | Full snapshot (ETag/gzip support) |
| `GET /api/user/:username` | Single user balance               |
| `GET /`                   | Balance tracker (main page)       |
| `GET /info`               | Analytics (claims, top holders)   |

## DUAT Node

```
http://121.99.241.161:5306   (atexoras.witness)
```

Protocol: `jsontoken: "duat"`, `prefix: "duat_"`, `VERSION: v1.4.5`.
